\section{Por que estudamos isso? (Contexto Profissional)}

Nesta unidade curricular, simulamos o \textbf{Pipeline de Desenvolvimento Moderno}. No mercado de trabalho, não basta apenas "saber programar", é preciso saber como entregar software de forma profissional, escalável e colaborativa. 

\begin{itemize}
    \item \textbf{Git:} Garante a integridade do código e o trabalho em equipe.
    \item \textbf{Flask:} Transforma lógica Python em serviços web reais.
    \item \textbf{Docker:} Resolve o problema de infraestrutura ("na minha máquina funciona").
\end{itemize}

\section{Versionamento de Código (Git \& GitHub)}

\begin{itemize}
    \item \texttt{git status}: O comando mais importante para saber o estado dos seus arquivos.
    \item \texttt{git add .} e \texttt{git commit -m "msg"}: O fluxo básico de registro de alterações.
    \item \texttt{git push origin main}: Sincronização com o servidor remoto.
    \item \texttt{git log --oneline}: Visualização rápida do histórico de versões.
\end{itemize}

\section{Orientação a Objetos (Python)}

\begin{boxRevisao}[Exemplo Prático OO]

\section{Desenvolvimento Web com Flask}

\begin{itemize}
    \item \textbf{Rotas:} Endereços que o servidor reconhece (\texttt{@app.route('/')}).
    \item \textbf{Jinja2:} \texttt{\{\{ variavel \}\}} exibe dados; \texttt{\{\% for ... \%\}} itera sobre listas.
    \item \textbf{GET vs POST:} Use \textbf{GET} para buscas e navegação; use \textbf{POST} para formulários de cadastro (via \texttt{request.form}).
\end{itemize}

\section{Virtualização e Docker}

\begin{boxDestaque}
\textbf{Comandos de Investigação:} Se o container subir mas o site não abrir, use \texttt{docker logs <nome>} para ver os erros do Python/Flask. Use \texttt{docker exec -it <nome> bash} para entrar no container.
\end{boxDestaque}

\begin{itemize}
    \item \textbf{Port Mapping (\texttt{-p 8080:5000}):} Mapeia a porta do Mac (8080) para a do Flask (5000).
    \item \textbf{Volumes (\texttt{-v \$(pwd):/app}):} Cria o túnel de edição em tempo real.
\end{itemize}

\section{Dockerfile e Orquestração}

\textbf{Docker Compose:}
\begin{itemize}
    \item \texttt{docker-compose up -d}: Sobe todos os serviços (Web + Banco).
    \item \texttt{docker-compose down}: Limpa todo o ambiente (remove containers e redes).
    \item \texttt{docker-compose logs -f}: Monitora os logs de todos os containers ao mesmo tempo.
\end{itemize}
